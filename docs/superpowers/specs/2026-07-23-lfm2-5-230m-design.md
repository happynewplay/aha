# LFM2.5-230M Support Design

## Goal

Support the local `D:\model_download\LFM2.5-230M` model as a distinct
LFM2.5 model size while preserving the existing LFM2.5-350M behavior.

## Design

Add a `WhichModel::LFM2_5_230M` variant with OpenAI model ID
`lfm2.5-230m`. Route it through the existing `LFM2_5` loader family and
declare the same supported artifacts as the 350M variant: Safetensors, GGUF,
and ONNX. The CLI uses the same LFM2.5 execution path and derives the default
request model name from the supplied load specification.

No second implementation module will be created. The shared LFM2.5 loader
already deserializes the model directory's `config.json`, so each model size
supplies its own hidden size, layer count, attention shape, and layer types.

## Validation

Add unit coverage that confirms the 230M variant has the expected model ID,
accepted artifacts, default artifact, and registry routing. Keep tests
independent of the local weight directory; existing integration tests remain
responsible for loading local model artifacts when available.
